# 扩展能力目录

本文件记录当前可用的扩展能力。每个扩展能力必须有一个同名 Markdown 文件，文件名格式为 `{name}.md`。

| 名称 | 文件 | 描述 |
|---|---|---|
| `init` | `init.md` | 当本地数据库不存在时，先向用户说明项目用途和能力，再询问是否开始使用。 |
| `install` | `install.md` | 帮用户检查和配置本地运行环境，并用最小案例测试确认项目可用。 |
| `dairy-work` | `dairy-work.md` | 帮用户创建每日任务清单自动化任务，按用户指定时间周期生成当天待办、日程和风险提醒。 |
| `backup` | `backup.md` | 备份和恢复当前用户数据，包括数据库、扩展目录和扩展目录登记。 |
| `update` | `update.md` | 从 GitHub 仓库检查并应用程序更新，更新前保护备份，更新后验证可用性。 |
