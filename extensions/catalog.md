# 扩展能力目录

本文件记录当前可用的扩展能力。每个扩展能力必须有一个同名 Markdown 文件，文件名格式为 `{name}.md`。

使用本目录时，先根据用户意图匹配下面的扩展名称；匹配后只读取对应文件，不要读取未匹配的扩展文件。

## `init`

文件：`init.md`

功能：当本地数据库不存在或项目尚未初始化时，向用户说明项目用途和能力，并确认是否开始使用。

触发场景：

- `assistant.py` 返回 `needs_init`。
- 用户表示首次启用、开始使用、初始化数据库或启用本项目。
- 用户询问数据库不存在、项目还不能记录或无法查询的原因。

## `install`

文件：`install.md`

功能：检查和配置本地运行环境，选择可用 Python 调用方式，并用最小案例验证项目可用。

触发场景：

- 用户要求安装、配置环境、检查环境或排查运行问题。
- `assistant.py doctor` 返回 `environment_error`。
- `assistant.py` 因 Python 命令不可用、解释器路径、权限或依赖问题无法启动。

## `daily-work`

文件：`daily-work.md`

功能：生成 DailyAssistant 每日日报，并支持配置每天或指定周期自动生成日报。

触发场景：

- 用户要求生成日报、晨报、每日简报或当天工作汇总。
- 用户要求查看当天任务、日程、逾期待办、已完成事项和待确认事项的汇总。
- 用户要求每天、定时、早上某个时间或按周期生成日报。
- 用户输入中包含“这是一个自动化任务”并要求生成日报。

## `backup`

文件：`backup.md`

功能：备份和恢复当前用户数据，包括数据库、扩展目录、扩展目录登记和事项文本导出。

触发场景：

- 用户要求备份、恢复、导出备份、创建保护副本或还原数据。
- 用户准备进行高风险维护，希望先保护当前数据。
- 用户询问如何迁移或保存当前 DailyAssistant 数据。

## `update`

文件：`update.md`

功能：从 GitHub 仓库检查并应用程序更新，更新前创建保护备份，更新后验证项目可用。

触发场景：

- 用户要求更新、升级、拉取新版、检查远程仓库或同步项目代码。
- 用户要求在保留现有数据的前提下更新 DailyAssistant。
- 用户询问当前项目是否有新版或如何应用新版程序。
