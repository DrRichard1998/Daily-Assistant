# init 初始化入口

## 1. 适用场景

当本项目不能正常进入记录、查询、完成、取消、聊天或其他扩展流程，并且原因与首次启动、数据库缺失或运行环境异常有关时，使用本扩展。

以下情况必须进入本扩展：

- `assistant.py` 返回 `status: needs_init`；
- `assistant.py doctor` 返回 `status: environment_error`；
- 用户要求安装、初始化、配置环境或排查运行环境；
- 当前确认 `data/assistant.sqlite` 不存在。

如果 `assistant.py` 命令自身无法运行，必须先读取错误文本判断原因。若原因是 `python` 命令不存在、Python 未加入 PATH、需要使用完整解释器路径，或沙箱需要授权执行本机 Python，则不要展示第 3 节的数据库缺失固定文案；应直接转入 `extensions/install.md` 的 Python 调用方式和环境排错流程。

进入本扩展后，不继续处理用户原始请求。无论用户原始请求是记录、查询、完成、取消、维护、聊天还是其他扩展请求，都先完成本扩展的说明和确认流程。

## 2. 职责边界

本扩展只负责三件事：

1. 告知用户当前项目还不能正常使用；
2. 逐字展示第 3 节的固定说明；
3. 询问用户是否开始使用本项目。

本扩展不执行以下操作：

- 不运行 `python .\assistant.py init`；
- 不运行 `extensions/install.md`；
- 不写入 SQLite；
- 不创建、删除或覆盖任何文件；
- 不恢复旧数据；
- 不继续执行用户原始请求。

只有用户明确确认开始使用本项目后，才转入第 5 节。

## 3. 固定展示内容

触发本扩展时，必须向用户逐字展示“固定文案开始”和“固定文案结束”之间的内容。不要改写、扩写、缩写或临时调整文案；不要放进 `text` 代码块或其他代码块；如需补充解释，只能放在固定文案之后。

“固定文案开始”和“固定文案结束”只是边界标记，不属于展示给用户的内容。

固定文案开始

当前本地数据库不存在，本项目还没有开始使用，或数据库已被删除。

DailyAssistant 是一个本地运行的任务和日程助理。它采用 LLM + Python + SQLite 的架构：LLM 负责理解你的输入，Python 程序负责校验、写入和查询，SQLite 负责在本机保存任务、日程和待确认事项。

本项目最大的优势是你不需要学习命令，也不需要把事情写成固定格式。你可以直接用自然语言输入，也可以发送语音或图片；例如，把邮件、通知、聊天记录或截图发给它，它会先转换和理解内容，再整理成任务、日程或待确认事项。

它的核心能力包括：
1. 记录：把自然语言整理为任务、日程或待确认事项。
2. 查询：查看今天、指定日期、时间范围、任务、日程和待确认事项。
3. 完成：把明确的任务标记为已完成。
4. 取消：把记错、识别错、取消或作废的事项软删除。
5. 聊天：与记录无关的问题不会写入数据库。
6. 扩展：通过 extensions 目录增加可选能力，例如安装检查和每日任务清单。

## 4. 用户确认规则

以下表达视为明确确认：

- “开始”
- “确认”
- “可以”
- “初始化”
- “开始使用”
- “是”
- “好”
- “继续”
- “按这个来”

以下情况不视为明确确认：

- 用户只是继续提出记录、查询、完成、取消或聊天请求；
- 用户询问项目细节或数据库缺失原因；
- 用户要求先恢复备份；
- 用户明确表示暂不使用或不想初始化；
- 用户表达含糊，例如“再说”“等等”“这是什么”。

用户未明确确认时，不得进入安装或初始化流程。

## 5. 用户确认后的流程

用户明确确认后，读取并执行：

```text
extensions/install.md
```

执行时完全遵守 `install.md`。本扩展不重复定义安装步骤，也不替代 `install.md` 做环境配置。

## 6. 用户未确认或拒绝时

用户未确认时，停止在本扩展，不初始化数据库，也不继续执行用户原始请求。

用户明确拒绝时，回复应简短说明：

```text
好的，暂不初始化。当前数据库不存在，因此记录、查询、完成和取消功能暂时不可用。
```

如果用户之后继续提出任何需要数据库的请求，仍然先进入本扩展，并再次展示第 3 节的固定说明。

## 7. 特殊情况

### 7.1 用户要求解释数据库缺失

先展示第 3 节固定说明，再补充简短解释。可以说明数据库可能尚未初始化、被删除、移动，或当前运行目录不正确。不要猜测具体原因。

### 7.2 用户要求恢复备份

先展示第 3 节固定说明，再说明当前数据库不存在。随后只问一个最小澄清问题：用户是要恢复已有备份，还是重新初始化一个空数据库。

### 7.3 用户要求安装或排查环境

先展示第 3 节固定说明。只有用户确认开始使用本项目后，才转入 `extensions/install.md`。

### 7.4 用户明确不想使用本项目

不运行 `install.md`，不初始化数据库。后续只要数据库仍不存在，任何依赖数据库的请求都继续进入本扩展。

## 8. 限制

- 本扩展不是安装脚本。
- 本扩展不是数据库恢复流程。
- 本扩展不修改 SQLite。
- 本扩展不创建、删除或覆盖任何文件。
- 本扩展不代替 `install.md` 做环境配置。
- 数据库被删除后，本扩展不能恢复旧数据；旧数据只能通过备份恢复。
