# dairy-work

## 目的

帮助用户创建“每日任务清单”的自动化任务。该扩展用于把用户的自然语言需求整理成可执行的自动化配置说明，让系统在指定时间定期生成当天任务清单。

## 触发条件

当用户表达以下意图时使用本扩展：

- 想每天固定时间生成任务清单；
- 想每天早上、晚上或某个时间查看当天安排；
- 想创建、调整或说明“每日任务清单”自动化；
- 提到“每日任务清单”“每天提醒我今天要做什么”“每天早上生成待办”等。

## 输出目标

该扩展不直接写入 `items`。它用于创建或描述自动化任务，自动化生成的内容应基于数据库查询结果。

每日任务清单应包含：

- 当天日程；
- 当天任务，来自 `query --date YYYY-MM-DD` 查询返回的 `tasks`；
- 已逾期且仍未完成的任务；
- 待确认事项；
- 时间冲突或临近风险。

## 默认规则

- 默认执行时间：每天 08:00。
- 默认时区：`Asia/Shanghai`。
- 默认语言：中文。
- 默认内容来源：只使用 `assistant.py` 查询 SQLite 的返回结果。
- 不推测数据库中不存在的任务或日程。
- 不列出数据库 ID，除非用户明确要求。

## 建议查询命令

生成当天任务清单时，优先使用：

```powershell
python .\assistant.py query --date YYYY-MM-DD
```

如果用户要求查看待确认事项：

```powershell
python .\assistant.py query --type reviews --status active
```

## 回复格式

创建或说明自动化任务时，回复应简洁说明：

```text
已创建每日任务清单自动化：
- 时间：每天 08:00
- 内容：当天日程、当天任务、逾期任务、待确认事项和风险提醒
- 来源：本地 SQLite 数据库
```

如果当前环境无法真正创建自动化任务，只说明已整理好自动化配置建议，不要假装已经创建。

## 限制

- 本扩展只定义每日任务清单自动化的行为，不负责底层调度器实现。
- 没有真实调度能力时，不要写入数据库表示自动化已创建。
- 不要修改 `data/assistant.sqlite`，除非用户明确要求记录一个相关任务或日程。
